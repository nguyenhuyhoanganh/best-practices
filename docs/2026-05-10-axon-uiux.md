## 1. Design Principles

| Nguyên tắc | Áp dụng |
|-----------|---------|
| **Clarity first** | Nội dung kỹ thuật phức tạp → layout rõ ràng, typography tốt |
| **Trust the admin flow** | Quy trình phê duyệt phải rõ ràng, không mơ hồ về trạng thái |
| **Reduce friction** | Submit form multi-step với auto-save, không mất dữ liệu |
| **Information density** | Developer user → chấp nhận nhiều thông tin hơn, không cần đơn giản hoá quá mức |

## 5. Component Specifications

### 5.1 TypeBadge

```
Props: type: BPType
Renders: pill badge với icon + label + màu theo type

SKILL_SET      → 🔧 Skill Set     (blue)
MCP_CONFIG     → 🔌 MCP Config    (green)
RULE_SET       → 📋 Rule Set      (orange)
AGENT_WORKFLOW → ⚡ Workflow       (violet)
```

### 5.2 StatusBadge

```
Props: status: BPStatus
Renders: pill badge với dot indicator + label

DRAFT          → ○ Draft          (gray)
PENDING_REVIEW → ● Pending Review  (amber)
UNDER_REVIEW   → ◐ Under Review    (blue)
PUBLISHED      → ✓ Published       (green)
REJECTED       → ✗ Rejected        (red)
```

### 5.3 RankingBadge

```
Props: score: number
Renders: ★ {score.toFixed(1)} với màu:
  score >= 100 → gold   (#F59E0B)
  score >= 50  → silver (#9CA3AF)
  score < 50   → bronze (#92400E) or hidden
```

### 5.4 StatusTimeline

```
Props: approvals: Approval[]
Renders: vertical timeline

  ● Submitted            (2026-05-08 09:00)
  ├─ ◐ Under Review      (2026-05-09 10:30, by Admin Huy)
  └─ ✓ Approved          (2026-05-09 14:00, by Admin Huy)
      "Nội dung tốt, approve!"
```

### 5.5 FileList

```
Props: files: BestPracticeFile[], bpId: string
Renders: list items mỗi file:
  [icon by mime] filename.ext    2.3 MB    [⬇ Download]

Download action → GET /api/v1/best-practices/:bpId/files/:fileId/download
→ follows 302 redirect automatically
```

### 5.6 AgentWorkflowEmbed

```
Props: workflowId: string, bpId: string
Renders:
  - Fetch workflow info từ /api/v1/agent-builder/workflows/:id
  - Hiển thị: tên workflow, mô tả, preview thumbnail (nếu có)
  - Button "Open in Agent Builder" → POST /use → redirect đến Agent Builder URL
  - Loading/error states
```

## 6. User Flows

### 6.1 First-time Login Flow

```
Land on AXon → [Sign in with Company SSO] → redirect SSO provider
→ SSO callback → JWT issued → redirect to Browse page
→ Welcome toast: "Chào mừng Huy Hoàng Anh!"
```

### 6.2 Submit Best Practice Flow

```
Click [+ Submit] → Multi-step form (Step 1-4)
→ Auto-save DRAFT on each step change
→ Step 4: Preview → [Submit for Review]
→ Toast: "Đã submit! Admin sẽ review trong 48h."
→ Redirect to My Submissions
→ Email to admins: "New best practice pending review"
```

### 6.3 Admin Review Flow

```
Admin sees badge "5 pending" in nav
→ /admin → Review Queue
→ Click [Review] → /admin/review/:id
  → Auto-set to UNDER_REVIEW
→ Review all content, download files to verify
→ Option A: [✓ Approve] → toast "Approved!" → redirect queue
            → contributor email: "Your submission was approved"
→ Option B: [✗ Reject] → modal với required comment
            → toast "Rejected with feedback" → redirect queue
            → contributor email: "Your submission needs changes: [comment]"
```

### 6.4 Find & Use Best Practice Flow

```
Browse page → search "confluence mcp" → filter MCP_CONFIG
→ See results → click card → Detail page
→ Read usage guide + install guide
→ Download config file → [⬇ Download]
→ (if WORKFLOW) → Open in Agent Builder
```

## 7. Empty States & Edge Cases

| Tình huống | UI |
|-----------|-----|
| Browse với không có kết quả | Illustration + "Chưa có best practice nào. Bạn có muốn đóng góp?" + [Submit] button |
| My Submissions rỗng | "Bạn chưa đăng ký best practice nào." + [+ Submit Now] |
| Admin queue rỗng | "Không có best practice nào chờ duyệt. " 🎉 |
| File upload lỗi (> 50MB) | Inline error dưới file input: "File vượt quá 50MB" |
| Agent Builder unavailable | "Không thể tải workflow preview. Thử lại sau." + retry button |
| Token expired (auto-refresh fail) | Toast: "Phiên đăng nhập hết hạn" → redirect login |

## 9. Accessibility

- All interactive elements có focus ring rõ ràng
- Color không phải cách duy nhất để truyền tải trạng thái (dùng icon + text kèm màu)
- Image alt text cho avatar
- Form labels gắn với inputs (`htmlFor`)
- Error messages gắn với field (`aria-describedby`)
- Loading states có `aria-busy`
