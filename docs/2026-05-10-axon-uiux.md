# AXon — UI/UX Design Document

**Version:** 2.1  
**Date:** 2026-05-12  
**Status:** Draft

---

## 1. Design Principles

| Nguyên tắc | Áp dụng |
|-----------|---------|
| **Clarity first** | Nội dung kỹ thuật phức tạp → layout rõ ràng, typography tốt |
| **Trust the review flow** | Quy trình review phải rõ ràng, không mơ hồ về trạng thái BP |
| **Reduce friction** | Form đăng ký multi-step với auto-save, không mất dữ liệu |
| **Information density** | Developer user → chấp nhận nhiều thông tin hơn, không đơn giản hoá quá mức |

## 2. Color System & Typography

### Color Palette

```
Primary:    #2563EB  (Blue 600)     — CTA, links, active states
Secondary:  #7C3AED  (Violet 600)  — EXTENSION badge
Success:    #16A34A  (Green 600)   — PUBLISHED, approved
Warning:    #D97706  (Amber 600)   — REQUESTED (pending review)
Danger:     #DC2626  (Red 600)     — REJECTED, CLOSED, delete actions
Neutral:    #6B7280  (Gray 500)    — secondary text, borders

Background: #F9FAFB  (Gray 50)    — page background
Card:       #FFFFFF               — card background
Border:     #E5E7EB  (Gray 200)   — card borders, dividers
Text:       #111827  (Gray 900)   — primary text
TextMuted:  #6B7280  (Gray 500)   — secondary text
```

### Type Badge Colors

```
WEB         → bg-blue-100    text-blue-700    (🌐)
TOOL        → bg-green-100   text-green-700   (🔧)
EXTENSION   → bg-violet-100  text-violet-700  (🧩)
```

### Status Badge Colors

```
REQUESTED   → bg-amber-100   text-amber-700   (● Requested)
REJECTED    → bg-red-100     text-red-700     (✗ Rejected)
PUBLISHED   → bg-green-100   text-green-700   (✓ Published)
```

> **Note:** Chỉ có 3 trạng thái BP. Action "close" (Supporter đóng BP đang published) chuyển status `PUBLISHED → REJECTED` và populate `close_reason`. Phân biệt close vs reject qua tooltip "Closed by supporter: <reason>" trên StatusBadge khi `close_reason != null`.

### Typography

```
Font family: Inter (sans-serif)
Heading 1:  32px, weight 700
Heading 2:  24px, weight 600
Heading 3:  18px, weight 600
Body:       16px, weight 400
Small:      14px, weight 400
Caption:    12px, weight 400
Code block: JetBrains Mono, 14px
```

## 5. Component Specifications

### 5.1 TypeBadge

```
Props: type: BPType
Renders: pill badge với icon + label + màu theo type

WEB       → 🌐 Web        (blue)
TOOL      → 🔧 Tool       (green)
EXTENSION → 🧩 Extension  (violet)
```

### 5.2 StatusBadge

```
Props: status: BPStatus, closeReason?: string
Renders: pill badge với dot indicator + label

REQUESTED → ● Requested   (amber)
REJECTED  → ✗ Rejected    (red) — nếu closeReason có giá trị, tooltip "Closed by supporter: <reason>"
PUBLISHED → ✓ Published   (green)
```

### 5.3 CreatorList

```
Props: creators: User[], maxShown: number = 1
Renders:
  [Avatar] Nguyen Van A       → nếu 1 creator
  [Avatar] Nguyen Van A +2    → nếu > 1 creator, tooltip shows all names
```

### 5.4 LikeButton

```
Props: bpId, initialCount, initialLiked
Renders: [❤ 32] hoặc [❤ 33] (toggled, màu đỏ khi liked)
Action: POST /best-practices/:id/like → cập nhật count optimistically
```

### 5.5 FilterPanel

```
Props: filters: FilterState, onChange: (filters) => void
Renders: 6 filter dimensions:
  - job (multi-select từ /api/v1/jobs)
  - ai_capability (multi-select từ /api/v1/ai-capabilities)
  - work_category (single-select, clears work khi đổi)
  - work (single-select, filtered by work_category)
  - type: WEB | TOOL | EXTENSION (single-select)
  - department (multi-select; options = SELECT DISTINCT users.department FROM users)
  - ai_tools_search (textbox; backend match `ai_tools_description ILIKE %query%`)

Active filter chips bên dưới, clickable để xoá từng filter.

Note: Department không còn là lookup table — options kéo từ distinct
users.department string. AI Tools đổi từ multi-select dropdown sang
textbox search vì AI tools giờ là free text trong ai_tools_description.
```

### 5.6 FeedbackSection

```
Props: bpId, feedbacks: Feedback[], currentUser: User
Renders:
  - List các feedback đã có (avatar, name, content, time)
  - Form để submit feedback mới (textarea + Submit button)
  - Chỉ hiển thị form nếu đã đăng nhập
  - Sau khi submit: optimistic update, clear textarea
```

### 5.7 ReviewPanel (AX Supporter)

```
Props: bp, onApprove, onReject, onClose
Renders tuỳ theo status hiện tại của BP:
  - REQUESTED: [Reject button (cần comment)] + [Approve button]
               ⚠ Nếu supporter là creator: cả 2 button disabled + warning message
  - PUBLISHED: [Close button (cần reason)]
  - REJECTED / CLOSED: read-only history

Comment textarea bắt buộc khi Reject; reason textarea bắt buộc khi Close.
```

### 5.8 ReviewHistory

```
Props: reviews: BpReview[]
Renders: vertical timeline

  ● Submitted        (2026-05-08 09:00, by Nguyen Van A)
  ├─ ✗ Rejected      (2026-05-09 10:30, by Admin Huy)
  │     "Thiếu installation guide"
  ├─ ● Resubmitted   (2026-05-09 14:00, by Nguyen Van A)
  └─ ✓ Approved      (2026-05-10 09:00, by Admin Huy)
```

## 6. User Flows

### 6.1 First-time Login

```
Land on AXon → [Sign in with Samsung CIP/AD] → redirect CIP/AD portal
→ CIP/AD callback → JWT issued → redirect to Library
→ Welcome toast: "Chào mừng Nguyen Van A!"
```

### 6.2 Register & Submit BP

```
Click [+ Register] → Step 1 (Basic Info)
→ Auto-save on step change
→ Step 2 (Content/Files — tuỳ type)
→ Step 3 (Installation Guide)
→ Step 4 (Preview) → [Submit]
→ Toast: "Đã submit! AX Supporter sẽ review sớm."
→ Redirect to My BPs
→ Email to AX Supporters: "New BP pending review"
```

### 6.3 AX Supporter Review Flow

```
→ /management → thấy BP REQUESTED → click [Review]
→ /management/:id/review → xem full detail (bao gồm key)
→ Option A: [✓ Approve]
  → toast "Published!" → redirect management
  → email to AX Creator: "Your BP was approved"
→ Option B: [✗ Reject] → nhập comment bắt buộc → confirm
  → toast "Rejected with feedback" → redirect management
  → email to AX Creator: "Your BP needs changes: [comment]"
```

### 6.4 AX Creator Edit Rejected BP

```
My BPs → thấy BP REJECTED với comment của supporter
→ click [Edit] → form pre-filled với data cũ
→ Sửa theo comment → click [Submit]
→ Status → REQUESTED, AX Supporter nhận thông báo
```

### 6.5 Edit Published BP (Re-review)

```
My BPs → thấy BP PUBLISHED → click [Edit]
→ Modal confirm: "Chỉnh sửa BP đã published sẽ ẩn BP và gửi lại để review. Tiếp tục?"
→ [Confirm] → form opens → edit → [Submit]
→ Status → REQUESTED, BP ẩn khỏi library
→ Toast: "BP đã được gửi lại để review."
```

### 6.6 Find & Use Best Practice

```
Library → search "confluence" → filter type=WEB
→ Xem card → click → Detail page
→ Đọc Installation Guide
→ [🌐 Open] (WEB) hoặc [⬇ Download] (TOOL/EXTENSION)
→ Like nếu thấy hữu ích → để lại feedback
```

### 6.7 Close BP (AX Supporter)

```
Management → thấy BP PUBLISHED → click [Close]
→ Modal: "Nhập lý do đóng BP" → [Close BP]
→ Status → CLOSED, BP ẩn khỏi library
→ email to AX Creator: "Your BP was closed: [reason]"
```

## 7. Empty States & Edge Cases

| Tình huống | UI |
|-----------|-----|
| Library không có kết quả | Illustration + "Chưa có best practice nào khớp. Thử thay đổi filter?" |
| My BPs rỗng | "Bạn chưa đăng ký best practice nào." + [+ Register Now] |
| Management queue rỗng (REQUESTED) | "Không có BP nào chờ review." 🎉 |
| File upload lỗi (> 50MB) | Inline error: "File vượt quá 50MB" |
| WEB content vượt 256 ký tự | Inline counter đỏ + lỗi khi submit |
| Self-approve attempt | Review panel: warning rõ ràng, button disabled |
| Token hết hạn (auto-refresh fail) | Toast "Phiên đăng nhập hết hạn" → redirect login |
| Feedback trên BP không phải PUBLISHED | Form ẩn, hiển thị "Chỉ có thể feedback BP đã published" |

## 9. Accessibility

- Tất cả interactive elements có focus ring rõ ràng
- Màu không phải cách duy nhất để truyền trạng thái (kết hợp icon + text + màu)
- Image alt text cho thumbnail và avatar
- Form labels gắn với inputs (`htmlFor`)
- Error messages gắn với field (`aria-describedby`)
- Loading states có `aria-busy`
- Modal confirm cho các action destructive (reject, close, edit-published)
