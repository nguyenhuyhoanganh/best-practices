# AXon — High-Level Design (HLD)

**Version:** 1.2  
**Date:** 2026-05-10  
**Status:** Approved

---

## 1. System Overview

AXon là một web platform nội bộ bao gồm 3 thành phần chính:

| Thành phần | Công nghệ | Vai trò |
|-----------|-----------|---------|
| Frontend | React 18 + TypeScript + Vite + TailwindCSS | SPA giao diện người dùng (Light Theme) |
| Backend | Java 21 + Spring Boot 3.5 | REST API, business logic, xác thực JWT |
| Infrastructure | PostgreSQL 16, Redis 7, MinIO | Dữ liệu, cache, object storage |

Hệ thống tích hợp trực tiếp với **Agent Builder API** để đồng bộ workflow.

---

## 3. Component Overview

### 3.1 Frontend (React SPA)

```
axon-frontend/
├── Auth Layer        — Login form (Username/Password), JWT storage
├── Browse Module     — Lọc theo Roles (Tags), Search, Trending section
├── Detail Module     — Xem chi tiết, TypeBadgeList, Download, Agent proxy
├── Submit Module     — Single-page form (Hỗ trợ multi-types selection)
├── MyWork Module     — Status tracking cá nhân
└── Admin Module      — Analytics Dashboard, Stats cards, Distribution charts
```

**State management:** Zustand (auth state), TanStack Query (server state/cache)  
**HTTP client:** Axios với interceptor tự động gắn JWT.

### 3.2 Backend (Spring Boot)

```
axon-backend/
├── auth/             — JWT, Login service (BCrypt), AuthController
├── user/             — User entity, role-based access control
├── bestpractice/     — CRUD, multi-type array support, role-based filtering
├── file/             — Upload/download qua MinIO
├── approval/         — Review flow & Analytics Service
├── usage/            — Logging, ranking scheduler
├── agentbuilder/     — Proxy client đến Agent Builder
└── notification/     — Mock notification service
```

### 3.3 Database (PostgreSQL)

5 bảng chính: `users`, `best_practices`, `best_practice_files`, `approvals`, `usage_logs`. 
Lưu ý quan trọng: `best_practices.types` và `best_practices.tags` đều là mảng chuỗi (`TEXT[]`).

## 4. Key Design Decisions

### 4.1 Traditional Authentication

Chuyển từ Mock SSO sang xác thực nội bộ để tăng tính ổn định:
- API `/auth/login` tiếp nhận Username/Password.
- Mật khẩu lưu dạng băm BCrypt.
- Cấp cặp JWT Token (Access/Refresh) cho Client.

### 4.2 Flexible Categorization & Discovery

Ưu tiên trải nghiệm khám phá theo chuyên môn:
- **Primary Filter:** Dựa trên Tags vai trò (Backend, Frontend...).
- **Technical Classification:** Một nội dung có thể thuộc nhiều loại hình kỹ thuật (Multi-types).

### 4.3 Admin Data Visualization

Dashboard tích hợp Analytics Service để tổng hợp dữ liệu thời gian thực:
- Tổng số tương tác (Views/Downloads).
- Tỷ lệ phân bổ nội dung theo bộ phận và loại hình kỹ thuật.
- Quy trình duyệt bài được tinh gọn với giao diện "Claim/Approve/Reject".
