## 1. System Overview

AXon là một web platform nội bộ bao gồm 3 thành phần chính:

| Thành phần | Công nghệ | Vai trò |
|-----------|-----------|---------|
| Frontend | React 18 + TypeScript + Vite + TailwindCSS | SPA giao diện người dùng |
| Backend | Java 21 + Spring Boot 3 | REST API, business logic, tích hợp |
| Infrastructure | PostgreSQL, Redis, MinIO | Dữ liệu, cache, file storage |

Hệ thống tích hợp với 2 external system:
- **SSO Provider** (pluggable — chưa xác định provider cụ thể)
- **Agent Builder** (existing REST API)

## 3. Component Overview

### 3.1 Frontend (React SPA)

```
axon-frontend/
├── Auth Layer        — SSO redirect, callback, JWT storage, route guard
├── Browse Module     — Danh sách, search, filter, trending
├── Detail Module     — Xem chi tiết BP, download, Agent Builder embed
├── Submit Module     — Form đa bước tạo/sửa best practice
├── MyWork Module     — Danh sách BP của user, status tracking
└── Admin Module      — Review queue, approve/reject, user management
```

**State management:** Zustand (auth state), TanStack Query (server state/cache)  
**HTTP client:** Axios với interceptor tự động gắn JWT và refresh token

### 3.2 Backend (Spring Boot)

```
axon-backend/
├── auth/             — JWT, SSO provider interface & mock
├── user/             — User entity, role management
├── bestpractice/     — CRUD, submit workflow, state machine
├── file/             — Upload/download qua MinIO
├── approval/         — Admin review flow
├── usage/            — Logging, ranking scheduler
├── agentbuilder/     — Proxy client đến Agent Builder
└── notification/     — Email notification service
```

### 3.3 Database (PostgreSQL)

5 bảng chính: `users`, `best_practices`, `best_practice_files`, `approvals`, `usage_logs`

### 3.4 Cache (Redis)

- Trending results (TTL 1h)
- Refresh token store (TTL 7d)

### 3.5 File Store (MinIO)

- Bucket: `axon-files`
- Object key pattern: `{best_practice_id}/{file_id}/{original_filename}`
- Download qua pre-signed URL (TTL 15 phút)

## 4. Key Design Decisions

### 4.1 SSO Pluggable Architecture

SSO provider chưa xác định → thiết kế interface:

```
SSOProvider (interface)
  ├── MockSSOProvider     (dev/test)
  ├── GoogleOAuthProvider (production option)
  ├── AzureADProvider     (production option)
  └── KeycloakProvider    (production option)
```

Spring profile `dev` dùng MockSSOProvider. Production chỉ cần implement interface và inject bean tương ứng.

### 4.2 Best Practice Status State Machine

```
DRAFT ──submit──► PENDING_REVIEW ──admin takes──► UNDER_REVIEW
  ▲                                                    │
  │                                               ┌────┴────┐
  └──────────────reject──────────────────────────►│ REJECTED│
                                                  └─────────┘
                                                       │ approve
                                                  ┌────▼────┐
                                                  │PUBLISHED│
                                                  └─────────┘
```

Chỉ DRAFT và REJECTED mới cho phép chỉnh sửa bởi contributor.

### 4.3 File Download Security

Files không bao giờ public. Mọi download đều qua:
1. Backend kiểm tra JWT + quyền xem BP
2. Backend tạo pre-signed URL từ MinIO (TTL 15 phút)
3. Client redirect đến pre-signed URL để download trực tiếp từ MinIO

### 4.4 Ranking Formula

```
usage_score = Σ (action_weight × time_decay_factor)

action_weight:
  VIEW           = 0.5
  DOWNLOAD       = 3.0
  WORKFLOW_USED  = 5.0

time_decay_factor (theo tuần):
  week 0 (hiện tại) = 1.0
  week 1            = 0.8
  week 2            = 0.64
  week N            = 0.8^N
```

Computed mỗi 1h bởi scheduled job, lưu vào `best_practices.usage_score`, cache trending vào Redis.

### 4.5 Agent Builder Integration

Backend đóng vai proxy:
- Không để frontend gọi trực tiếp Agent Builder (tránh CORS, tập trung auth)
- Backend gọi Agent Builder với service-to-service credential
- Log `WORKFLOW_USED` khi user dùng workflow

## 5. Security Architecture

```
Request flow:
Browser → [JWT in Authorization header] → Spring Security Filter
  → JwtAuthenticationFilter (validate JWT)
  → SecurityContext (set current user)
  → Controller → Service (check ownership/role)
  → Response
```

**Token flow:**
```
SSO Login:
  Browser → /auth/sso/login → redirect to SSO Provider
  SSO Provider → /auth/sso/callback (with code)
  Backend → exchange code → get user info → upsert user → issue JWT pair
  JWT: { access_token (15m), refresh_token (7d) }

Token refresh:
  Browser → /auth/refresh (with refresh_token)
  Backend → validate refresh_token in Redis → issue new access_token
```

## 6. Deployment Architecture

### Dev (Docker Compose)

```yaml
services:
  axon-backend:  # Spring Boot, port 8080
  axon-frontend: # Vite dev server, port 5173
  postgres:      # PostgreSQL 16, port 5432
  redis:         # Redis 7, port 6379
  minio:         # MinIO, port 9000 (API) + 9001 (console)
```

### Production (Kubernetes)

```
Deployment: axon-backend   (2 replicas, HPA)
Deployment: axon-frontend  (2 replicas, Nginx)
Service:    axon-backend-svc
Service:    axon-frontend-svc
Ingress:    axon.company.internal
ConfigMap:  axon-config
Secret:     axon-secrets (DB creds, JWT secret, MinIO creds, SSO client secret)
```

External services trên production:
- PostgreSQL: managed service hoặc on-prem
- Redis: managed service hoặc on-prem
- MinIO: on-prem hoặc S3 compatible

## 7. API Communication

**Protocol:** REST over HTTPS  
**Format:** JSON (request + response)  
**Auth:** `Authorization: Bearer <access_token>`  
**Versioning:** Path-based `/api/v1/`  
**Error format:**
```json
{
  "error": "RESOURCE_NOT_FOUND",
  "message": "Best practice not found",
  "timestamp": "2026-05-10T10:00:00Z"
}
```

## 8. Non-Functional Design

| Concern | Approach |
|---------|---------|
| Scalability | Backend stateless (JWT, không dùng server session) → scale horizontal |
| Caching | Redis cho trending (hot data), TanStack Query cho client-side cache |
| File storage | MinIO tách biệt khỏi app server — scale độc lập |
| Observability | Spring Actuator + structured logging (JSON) |
| DB migrations | Flyway — auto-run khi startup, version-controlled |
| Testing | JUnit 5 + Mockito (unit), Testcontainers (integration), Playwright (E2E) |

## 9. Technology Stack Summary

| Layer | Technology | Lý do chọn |
|-------|-----------|------------|
| FE Framework | React 18 + TypeScript | Ecosystem lớn, type-safe, team quen thuộc |
| FE Build | Vite | Build nhanh hơn CRA, HMR tốt |

| BE Framework | Spring Boot 3 + Java 21 | Enterprise-grade, team Java mạnh, virtual threads |
| BE Auth | Spring Security OAuth2 | Native integration, pluggable provider |
| DB | PostgreSQL 16 | JSONB cho external_links, arrays cho tags, mature |
| ORM | Spring Data JPA + Hibernate | Standard, ít boilerplate với Spring Boot |
| Migration | Flyway | Versioned, auto-run, team workflow tốt |
| Cache | Redis 7 | Reliable, supports TTL, cộng đồng lớn |
| File Store | MinIO | S3-compatible, self-hosted, dễ thay bằng S3 |
| Container | Docker + Docker Compose | Dev environment consistency |
| CI/CD | GitHub Actions | Tích hợp sẵn với GitHub repo |
