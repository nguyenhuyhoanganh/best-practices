## 1. System Overview

AXon là một web platform nội bộ bao gồm 3 thành phần chính:

| Thành phần | Công nghệ | Vai trò |
|-----------|-----------|---------|
| Frontend | React 18 + TypeScript + Vite + TailwindCSS | SPA giao diện người dùng |
| Backend | Java 21 + Spring Boot 3 | REST API, business logic, tích hợp |
| Infrastructure | PostgreSQL, Redis, MinIO | Dữ liệu, cache, file storage |

Hệ thống tích hợp với 2 external system:
- **SSO Provider** (pluggable — chưa xác định provider cụ thể)
- **Agent Builder** (existing REST API)

## 3. Component Overview

### 3.1 Frontend (React SPA)

```
axon-frontend/
├── Auth Layer        — SSO redirect, callback, JWT storage, route guard
├── Browse Module     — Danh sách, search, filter, trending
├── Detail Module     — Xem chi tiết BP, download, Agent Builder embed
├── Submit Module     — Form đa bước tạo/sửa best practice (hỗ trợ multi-types)
├── MyWork Module     — Danh sách BP của user, status tracking
└── Admin Module      — Review queue, approve/reject, user management
```

**State management:** Zustand (auth state), TanStack Query (server state/cache)  
**HTTP client:** Axios với interceptor tự động gắn JWT.

### 3.2 Backend (Spring Boot)

```
axon-backend/
├── auth/             — JWT, SSO provider interface & mock
├── user/             — User entity, role management
├── bestpractice/     — CRUD, submit workflow, multi-type support
├── file/             — Upload/download qua MinIO
├── approval/         — Admin review flow
├── usage/            — Logging, ranking scheduler
├── agentbuilder/     — Proxy client đến Agent Builder
└── notification/     — Mock notification service
```

### 3.3 Database (PostgreSQL)

5 bảng chính: `users`, `best_practices`, `best_practice_files`, `approvals`, `usage_logs`. 
Lưu ý: `best_practices.types` là một mảng chuỗi (`TEXT[]`).

## 4. Key Design Decisions

### 4.1 SSO Pluggable Architecture

SSO provider chưa xác định → thiết kế interface:

```
SSOProvider (interface)
  ├── MockSSOProvider     (dev/test)
  ├── GoogleOAuthProvider (production option)
  └── KeycloakProvider    (production option)
```

### 4.2 Best Practice Status State Machine

```
DRAFT ──submit──► PENDING_REVIEW ──admin takes──► UNDER_REVIEW
  ▲                                                    │
  │                                               ┌────┴────┐
  └──────────────reject──────────────────────────►│ REJECTED│
                                                  └─────────┘
                                                       │ approve
                                                  ┌────▼────┐
                                                  │PUBLISHED│
                                                  └─────────┘
```

### 4.3 Ranking Formula

```
usage_score = Σ (action_weight × time_decay_factor)

action_weight:
  VIEW           = 0.5
  DOWNLOAD       = 3.0
  WORKFLOW_USED  = 5.0
```

### 4.4 Flexible Categorization

Hệ thống phân tách rõ:
- **Types:** Bản chất kỹ thuật (MCP, Skill, Hooks, Rule, Workflow, Prompt, Tool). Cho phép chọn nhiều.
- **Tags:** Phạm vi công việc (backend, frontend, devops, ba, pm...).
