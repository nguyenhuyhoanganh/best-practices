## 1. System Overview

AXon là một web platform nội bộ bao gồm 3 thành phần chính:

| Thành phần | Công nghệ | Vai trò |
|-----------|-----------|---------|
| Frontend | React 18 + TypeScript + Vite + TailwindCSS | SPA giao diện người dùng |
| Backend | Java 21 + Spring Boot 3 | REST API, business logic, xác thực |
| Infrastructure | PostgreSQL, Redis, MinIO | Dữ liệu, session/cache, file storage |

Hệ thống tích hợp với 1 external system:
- **CIP/AD** (Samsung Corporate Identity Provider — SSO nội bộ)

## 3. Component Overview

### 3.1 Frontend (React SPA)

```
axon-frontend/
├── Auth Layer        — CIP/AD redirect, callback, JWT storage, route guard
├── Library Module    — Browse, search, filter, sort, trending
├── Detail Module     — Xem chi tiết BP, download, like, feedback
├── Register Module   — Form đa bước tạo/sửa best practice
├── MyBPs Module      — Danh sách BP của AX Creator, analytics
├── Management Module — Review queue, approve/reject/close (AX Supporter)
├── Dashboard Module  — Monitoring stats (AX Supporter)
└── Admin Module      — User management (Admin)
```

**State management:** Zustand (auth state), TanStack Query (server state/cache)  
**HTTP client:** Axios với interceptor tự động gắn JWT và refresh token

### 3.2 Backend (Spring Boot)

```
axon-backend/
├── auth/             — CIP/AD provider interface & mock, JWT
├── user/             — User entity, role management
├── bestpractice/     — CRUD, submit workflow, state machine
├── file/             — Upload/download qua MinIO, pre-signed URL
├── management/       — AX Supporter: approve/reject/close, review history
├── interaction/      — Like, download tracking, feedback
├── lookup/           — Job, AI capability, work, department, AI tool APIs
├── analytics/        — BP analytics cho AX Creator
├── dashboard/        — Dashboard stats cho AX Supporter
└── notification/     — Email notification service
```

### 3.3 Database (PostgreSQL)

**Lookup tables (5):** `jobs`, `ai_capabilities`, `work_categories`, `works`, `departments`, `ai_tools`

**Core tables (3):** `users`, `best_practices`, `bp_files`

**Junction tables (5):** `bp_creators`, `bp_jobs`, `bp_ai_capabilities`, `bp_ai_tools`, `bp_departments`

**Interaction tables (4):** `bp_likes`, `bp_feedback`, `bp_downloads`, `bp_reviews`

### 3.4 Cache (Redis)

- Session / refresh token store (TTL 7d)
- Rate limiting counters
- Dashboard stats cache (TTL 15 phút)

### 3.5 File Store (MinIO)

- Bucket: `axon-files`
- Object key pattern: `{best_practice_id}/{file_id}/{original_filename}`
- Download qua pre-signed URL (TTL 15 phút)
- Chỉ áp dụng với BP type TOOL và EXTENSION

## 4. Key Design Decisions

### 4.1 CIP/AD Pluggable Architecture

CIP/AD là provider chính nhưng thiết kế theo interface để linh hoạt:

```
SSOProvider (interface)
  ├── MockSSOProvider     (dev/test — trả về hardcoded user)
  └── CIPADProvider       (production — Samsung CIP/AD)
```

Spring profile `dev` dùng MockSSOProvider. Production inject CIPADProvider.

### 4.2 BP Status State Machine

```
REQUESTED ─── approve ──► PUBLISHED ─── close ──► CLOSED
    ▲               │
    │          [Creator edit]
  resubmit          │
    │               ▼
 REJECTED ◄── reject ── REQUESTED
```

Transition rules:
- Chỉ AX Supporter mới được approve/reject/close
- AX Supporter không thể approve BP mà mình là creator
- Edit PUBLISHED BP → auto-chuyển REQUESTED (ẩn khỏi library)
- CLOSED không cho phép chỉnh sửa hay reopen

### 4.3 File Download Security

Files không bao giờ public. Mọi download đều qua:
1. Backend kiểm tra JWT + quyền xem BP (must be PUBLISHED hoặc owner/supporter)
2. Backend tạo pre-signed URL từ MinIO (TTL 15 phút)
3. Backend ghi log DOWNLOAD (bp_downloads table)
4. Client redirect đến pre-signed URL

### 4.4 Key Field Visibility

Trường `key_value` trong best_practices là nhạy cảm:
- **Ẩn** với User thông thường (không trả về trong API response)
- **Hiển thị** với: AX Creator (là owner/creator của BP), AX Supporter, Admin
- Backend kiểm tra role và ownership trước khi include field trong response

### 4.5 Like Mechanism

- Toggle: like lần 1 → thêm; like lần 2 → bỏ (unlike)
- Unique constraint `(bp_id, user_id)` trong table `bp_likes`
- `like_count` trong `best_practices` được update atomic khi toggle
- API trả về cả `like_count` và `is_liked_by_current_user`

### 4.6 Multiple Creators

- Một BP có thể có nhiều creator qua junction table `bp_creators`
- Card hiển thị: tên creator đầu tiên + "+N others" nếu N > 1
- Tất cả creator đều có thể edit BP (nếu status cho phép)
- Self-approve check: AX Supporter bị chặn nếu user_id của họ có trong `bp_creators` của BP đó

### 4.7 Taxonomy Management

Các bảng lookup (job, ai_capability, work_category, work, department, ai_tool) là master data:
- Admin quản lý qua Admin API
- Frontend dùng để populate dropdowns khi đăng ký BP và filter
- Seed data ban đầu theo giá trị mẫu từ requirements

## 5. Security Architecture

```
Request flow:
Browser → [JWT in Authorization header] → Spring Security Filter
  → JwtAuthenticationFilter (validate JWT)
  → SecurityContext (set current user + role)
  → Controller → Service (check role + ownership)
  → Response (mask key_value nếu không đủ quyền)
```

**Token flow:**
```
CIP/AD Login:
  Browser → /auth/login → redirect to CIP/AD
  CIP/AD  → /auth/callback (with code)
  Backend → exchange code → get user info → upsert user → issue JWT pair
  JWT: { access_token (15m), refresh_token (7d) }

Token refresh:
  Browser → /auth/refresh (with refresh_token)
  Backend → validate refresh_token in Redis → issue new access_token
```

## 6. Deployment Architecture

### Dev (Docker Compose)

```yaml
services:
  axon-backend:  # Spring Boot, port 8080
  axon-frontend: # Vite dev server, port 5173
  postgres:      # PostgreSQL 16, port 5432
  redis:         # Redis 7, port 6379
  minio:         # MinIO, port 9000 (API) + 9001 (console)
```

### Production (Kubernetes)

```
Deployment: axon-backend   (2 replicas, HPA)
Deployment: axon-frontend  (2 replicas, Nginx)
Service:    axon-backend-svc
Service:    axon-frontend-svc
Ingress:    axon.company.internal
ConfigMap:  axon-config
Secret:     axon-secrets (DB creds, JWT secret, MinIO creds, CIP/AD client secret)
```

External services trên production:
- PostgreSQL: managed service hoặc on-prem
- Redis: managed service hoặc on-prem
- MinIO: on-prem hoặc S3 compatible

## 7. API Communication

**Protocol:** REST over HTTPS  
**Format:** JSON (request + response)  
**Auth:** `Authorization: Bearer <access_token>`  
**Versioning:** Path-based `/api/v1/`  
**Error format:**
```json
{
  "error": "RESOURCE_NOT_FOUND",
  "message": "Best practice not found",
  "timestamp": "2026-05-11T10:00:00Z"
}
```

## 8. Non-Functional Design

| Concern | Approach |
|---------|---------|
| Scalability | Backend stateless (JWT) → scale horizontal |
| Caching | Redis cho dashboard stats (15m TTL), TanStack Query cho client-side |
| File storage | MinIO tách biệt khỏi app server — scale độc lập |
| Observability | Spring Actuator + structured logging (JSON) |
| DB migrations | Flyway — auto-run khi startup, version-controlled |
| Testing | JUnit 5 + Mockito (unit), Testcontainers (integration), Playwright (E2E) |
